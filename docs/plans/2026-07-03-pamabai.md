# PapersMadeByAI (PaMaBAI) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A public document manager at `papersmadebyai.tokenstree.eu` where anyone can upload AI-authored papers (PDF or LaTeX), plus an online scientific journal ("The PaMaBAI Journal") whose issues are curated from those papers.

**Architecture:** Single lean FastAPI app (async, SQLite + FTS5, uploads on disk) with server-rendered Jinja2 templates and hand-crafted CSS (no build chain). nginx vhost + Let's Encrypt on worker1 (213.165.78.202, where the subdomain's A record already points). systemd service, port 8096.

**Tech Stack:** Python 3.11+, FastAPI, uvicorn, aiosqlite, Jinja2, python-multipart; nginx + certbot; no JS framework (progressive enhancement only).

## Global Constraints

- All UI copy in **English**.
- Uploads: `.pdf`, `.tex`, `.zip` (LaTeX bundle); max **25 MB**; extension + magic-bytes check; stored under `/opt/pamabai/uploads/{paper_id}/`.
- Every paper MUST declare **AI provenance**: `ai_models` (which model(s) wrote it) and `human_role` (supervision level). This is the site's signature field and is displayed on every card.
- Admin actions (create issue, add article to issue, unpublish) require `X-Admin-Token` (env `PAMABAI_ADMIN_TOKEN`), never hardcoded.
- Server RAM budget ≤ 150 MB (worker1 is shared with hibrid, androidwars, hnreviewer, tt worker).
- No CDN dependencies: all CSS/fonts self-contained (system font stacks).
- DB file `/opt/pamabai/pamabai.db`; WAL mode; single uvicorn worker.

## Part A — What is a scientific journal made only of AI-authored papers?

**Concept.** Every article is researched, executed and written by AI agents; humans own scope, audit and accountability. The journal's credibility cannot come from author reputation (the authors are models), so it must come from **radical provenance**: every claim traceable to raw data and code, every paper carrying a machine-readable disclosure of which models did what.

**Editorial policy (v1, enforced by the platform):**
1. **Mandatory AI-provenance block** — model names/versions + orchestration method + human role. Stored as structured metadata, shown on every article page.
2. **Artifact-or-it-didn't-happen** — an article without a public code/data link is a *preprint* forever; only artifact-backed papers can enter an issue.
3. **AI peer review, disclosed** — referee reports are produced by independent AI reviewer agents (different model/session from the authors) and published alongside the article (open review, TMLR-style).
4. **Honesty over polish** — negative results, corrections and contamination notes are first-class content (the hibrid paper's "data integrity" note is the house style).
5. **Human accountability** — each paper lists a human owner responsible for the claims; the journal is transparent that it is an experiment in AI-made science, not a claim of peer-reviewed authority. No fake ISSN/impact factor; numbering is internal (Vol. 1, 2026).

**Why it can work:** AI-made papers are exploding on arXiv with disclosure hidden in footnotes; a venue that makes provenance the *point* — instead of something to hide — fills a real gap (documentation of applied AI systems built by agents, evaluations, tooling reports). Weakness to state openly: AI referees share failure modes with AI authors; mitigation is artifact-first review and public reviews.

**Issue #1 (bootstrap):** seven system/evaluation papers from the TokensTree ecosystem, all AI-made, all with public artifacts: hibrid (router evaluation), tokenstree.com (agent social platform), tokenstransfer, tokenstranslate, androidwars (AI-only RTS), tokenstree.es (trading predictor), tokenstree.eu (AI newsletter pipeline).

## Part B — File Structure

```
pamabai/
├── app/
│   ├── main.py          # FastAPI app, lifespan (DB init), routes wiring
│   ├── db.py            # aiosqlite helpers, schema DDL, FTS5 triggers
│   ├── models.py        # pydantic schemas (PaperIn, Paper, Issue, Article)
│   ├── papers.py        # upload/list/detail/download endpoints + validation
│   ├── journal.py       # issues + articles endpoints (admin-gated writes)
│   ├── security.py      # admin token dep, upload validation (size/magic)
│   └── templates/
│       ├── base.html    # layout, header, footer, CSS inline <style>
│       ├── index.html   # landing: hero, latest papers, journal teaser
│       ├── papers.html  # browse/search list
│       ├── paper.html   # detail: metadata, provenance block, PDF viewer
│       ├── upload.html  # upload form (PDF/LaTeX)
│       ├── journal.html # journal home: issues list, editorial policy
│       └── issue.html   # issue TOC — looks like a real journal issue
├── tests/test_app.py    # httpx AsyncClient tests over the app
├── requirements.txt
├── deploy/
│   ├── pamabai.service  # systemd unit
│   └── nginx.conf       # vhost for papersmadebyai.tokenstree.eu
└── docs/plans/2026-07-03-pamabai.md   # this plan
```

**DB schema:**
```sql
papers(id TEXT PK, title TEXT, authors TEXT, abstract TEXT, keywords TEXT,
       ai_models TEXT, human_role TEXT, artifact_url TEXT, license TEXT,
       kind TEXT CHECK(kind IN ('pdf','latex')), filename TEXT, size_bytes INT,
       status TEXT DEFAULT 'preprint',  -- preprint | published | hidden
       created_at TEXT)
papers_fts(FTS5: title, authors, abstract, keywords, content=papers)
issues(id INTEGER PK, volume INT, number INT, title TEXT, editorial TEXT,
       published_at TEXT)
articles(issue_id INT, paper_id TEXT, position INT, pages TEXT,
         PRIMARY KEY(issue_id, paper_id))
```
